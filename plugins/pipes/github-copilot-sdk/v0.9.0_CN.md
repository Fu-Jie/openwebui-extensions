## 🚀 v0.9.0 更新内容

### 1. 工作区自定义工具 — 在任何地方定义工具

本次更新引入了**工作区自定义工具**，一个全新的框架，让你在工作区中定义自定义工具，无需修改插件代码。

**核心特性**:
- 在 `.copilot-skills/` 目录中使用 `@define_tool` 装饰器定义工具
- Copilot SDK 自动发现和注册
- 完整支持工具参数、异步操作和类型提示
- 提供示例模板快速开始

**工作原理**:

1. 在工作区根目录创建 `.copilot-skills/` 目录
2. 使用 `@define_tool` 装饰器定义工具（与官方 SDK 相同）
3. SDK 在初始化时自动发现工具
4. 工具立即可用于你的 Copilot Agent

**示例**:

```python
from copilot import define_tool

@define_tool(description="根据 ID 获取用户信息")
async def get_user_by_id(user_id: str) -> dict:
    """获取用户详细信息。"""
    return {"id": user_id, "name": "张三"}

@define_tool(description="异步处理数据")
async def process_data(data: list[str]) -> dict:
    """处理数据项列表。"""
    return {"processed_count": len(data), "items": data}
```

查看 [workspace_skills_example.py](workspace_skills_example.py) 获取完整模板，包含 3 个实用示例。

### 2. 自动发现机制

SDK 现在在初始化时扫描 `{workspace}/.copilot-skills/` 目录并：
- 自动导入所有有效的 Python 模块
- 发现并注册所有 `@define_tool` 装饰的函数
- 错误处理和详细日志记录

无需手动注册 — 只需编写工具，它就会自动工作。

### 3. 增强的配置选项

新增 Valve 参数：
- **`ENABLE_WORKSPACE_TOOLS`**（默认: `True`）— 切换工作区自定义工具自动发现开关。启用时，插件会在 `{workspace}/.copilot-skills/` 搜索自定义工具。

### 4. 完整文档支持

- 更新的 README 包含详细的工作区自定义工具使用指南
- 包含函数示例（数学计算器、文本处理器、数据分析工具）
- 现有部署的迁移说明
- 示例实现模板

## 🔄 升级指南

**从 v0.8.0 升级到 v0.9.0**:

1. **无破坏性变更** — 所有现有功能保持不变
2. **新增 Valve**: `ENABLE_WORKSPACE_TOOLS`（默认: `True`）
   - 切换工作区自定义工具开关
   - 默认启用，所以如果目录存在，自定义工具会自动被发现
3. **创建自定义工具**:
   - 在工作区根目录创建 `.copilot-skills/` 目录
   - 使用 `@define_tool` 装饰器定义工具
   - 工具立即可用于你的 Copilot Agent

**工作区目录结构示例**:
```
your-workspace/
├── .copilot-skills/
│   ├── search_tools.py
│   ├── data_processors.py
│   └── README.md
└── ... (你的其他文件)
```

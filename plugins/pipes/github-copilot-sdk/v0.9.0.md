## 🚀 What's New in v0.9.0

### 1. Workspace Skills — Define Custom Tools Anywhere

This release introduces **Workspace Skills**, a new framework for defining custom tools in your workspace without modifying the plugin itself.

**Key Features**:
- Define tools using the `@define_tool` decorator in files within `.copilot-skills/` directory
- Automatic discovery and registration by the Copilot SDK
- Full support for tool parameters, async operations, and type hints
- Example template provided for quick start

**How It Works**:

1. Create a `.copilot-skills/` directory in your workspace root
2. Define tools using the `@define_tool` decorator (same as in the official SDK)
3. The SDK auto-discovers tools on plugin initialization
4. Tools are immediately available in your Copilot agents

**Example**:

```python
from copilot import define_tool

@define_tool(description="Retrieve user information")
async def get_user_by_id(user_id: str) -> dict:
    """Get user details by ID."""
    return {"id": user_id, "name": "John Doe"}

@define_tool(description="Process data asynchronously")
async def process_data(data: list[str]) -> dict:
    """Process a list of data items."""
    return {"processed_count": len(data), "items": data}
```

See [workspace_skills_example.py](workspace_skills_example.py) for a complete template with 3 working examples.

### 2. Auto-Discovery Mechanism

The SDK now scans `{workspace}/.copilot-skills/` during initialization and:
- Automatically imports all valid Python modules
- Discovers and registers `@define_tool` decorated functions
- Handles import errors gracefully with detailed logging

No manual registration needed — just write your tool and it works.

### 3. Enhanced Configuration

New Valve added:
- **`ENABLE_WORKSPACE_SKILLS`** (default: `True`) — Toggle workspace skills auto-discovery on/off. When enabled, the plugin searches `{workspace}/.copilot-skills/` for custom tools.

### 4. Comprehensive Documentation

- Updated README with detailed Workspace Skills guide
- Included function examples (Math Helper, Text Processor, Data Analyzer)
- Migration notes for existing deployments
- Example implementation template

## 🔄 Migration Guide

**Upgrading from v0.8.0 to v0.9.0**:

1. **No breaking changes** — All existing functionality remains intact
2. **New Valve**: `ENABLE_WORKSPACE_SKILLS` (default: `True`)
   - Toggle Workspace Skills on/off
   - Default is ON, so custom tools are auto-discovered if the directory exists
3. **Creating Custom Tools**:
   - Create `.copilot-skills/` directory in your workspace root
   - Define tools using `@define_tool` decorator
   - Tools are immediately available to your Copilot agents

**Example Workspace Structure**:
```
your-workspace/
├── .copilot-skills/
│   ├── search_tools.py
│   ├── data_processors.py
│   └── README.md
└── ... (your other files)
```

## 🐞 Known Limitations

- Tools must use the `@define_tool` decorator from the SDK
- External dependencies must be installed in the OpenWebUI environment
- Tool names are automatically converted to snake_case
